# ☕ Java Basics Overview

## 1️⃣ Platform Independent

* Java code can run on **any operating system** using the **JVM (Java Virtual Machine)**.
* Famous Java motto:

> **"Write Once, Run Anywhere (WORA)"**

**Flow:**

```
Java Source Code (.java)
        ↓
     Compiler
      (javac)
        ↓
     Bytecode (.class)
        ↓
        JVM
        ↓
Runs on Windows / Linux / macOS
```

---

## 2️⃣ Object-Oriented Language (OOP)

Java is built around **objects and classes**.

### Core OOP Concepts

| Concept           | Description                                   |
| ----------------- | --------------------------------------------- |
| **Encapsulation** | Wrapping data and methods into a single unit  |
| **Inheritance**   | One class can inherit properties from another |
| **Polymorphism**  | Same method behaving differently              |
| **Abstraction**   | Hiding implementation details                 |

---

## 3️⃣ Compiled + Interpreted Language

Java uses **both compilation and interpretation**.

| Step            | Description                                             |
| --------------- | ------------------------------------------------------- |
| **Compilation** | `.java` source file compiled into **bytecode (.class)** |
| **Execution**   | JVM interprets and executes the bytecode                |

```
.java file
   ↓
javac compiler
   ↓
.class bytecode
   ↓
JVM executes
```

✔ Gives both **speed** (compiled) and **portability** (interpreted)

---

## 4️⃣ Java File Extensions

| File Type     | Extension | Description                    |
| ------------- | --------- | ------------------------------ |
| Source Code   | `.java`   | Human-readable Java code       |
| Compiled Code | `.class`  | Bytecode generated by compiler |

Example:

```
HelloWorld.java
      ↓
javac HelloWorld.java
      ↓
HelloWorld.class
```

# Primitive Data Types Size

| Data Type    | Size (Bytes) | Size (Bits) | Example               |
| ------------ | ------------ | ----------- | --------------------- |
| **byte**     | 1 byte       | 8 bits      | `byte a = 10;`        |
| **short**    | 2 bytes      | 16 bits     | `short b = 200;`      |
| **int**      | 4 bytes      | 32 bits     | `int c = 5000;`       |
| **long**     | 8 bytes      | 64 bits     | `long d = 100000L;`   |
| **float**    | 4 bytes      | 32 bits     | `float e = 5.5f;`     |
| **double**   | 8 bytes      | 64 bits     | `double f = 3.14159;` |
| **char**     | 2 bytes      | 16 bits     | `char g = 'A';`       |
| **boolean*** | ~1 byte*     | 8 bits*     | `boolean h = true;`   |

*Boolean size depends on JVM implementation.

---

# Range Formula for Integer Data Types

To calculate the range of a signed integer:

```
Minimum Value = -2^(n-1)
Maximum Value = 2^(n-1) - 1
```

Where:

```
n = number of bits
```

Example for **byte (8 bits)**:

```
Minimum = -2^(8-1) = -128
Maximum = 2^(8-1) - 1 = 127
```

Range:

```
-128 to 127
```

# 🧠 Object-Oriented Programming (OOP) Concepts in Java

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around **objects and classes**.

---

## 📌 Main OOP Concepts

| Concept       | Description                                                       |
| ------------- | ----------------------------------------------------------------- |
| Encapsulation | Wrapping data and methods into a single unit                      |
| Inheritance   | One class acquiring properties of another class                   |
| Polymorphism  | One method behaving in multiple ways                              |
| Abstraction   | Hiding implementation details and showing only essential features |

---

# 1️⃣ Encapsulation

Encapsulation means **hiding data and allowing access through methods**.

## Example

```java
class Student {

    private String name;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

public class Main {
    public static void main(String[] args) {

        Student s = new Student();
        s.setName("Sanmark");

        System.out.println(s.getName());
    }
}
```

### Key Points

* Variables are **private**
* Access through **getter and setter methods**
* Provides **data security**

---

# 2️⃣ Inheritance

Inheritance allows a class to **reuse properties and methods of another class**.

## Example

```java
class Animal {
    void eat() {
        System.out.println("Animal is eating");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog is barking");
    }
}

public class Main {
    public static void main(String[] args) {

        Dog d = new Dog();
        d.eat();
        d.bark();
    }
}
```

### Key Points

* Uses **extends keyword**
* Promotes **code reusability**
* Supports **hierarchical relationships**

---

# 3️⃣ Polymorphism

Polymorphism means **one method with different behaviors**.

## Example (Method Overloading)

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

### Key Points

* Same method name
* Different parameters
* Improves **flexibility**

---

# 4️⃣ Abstraction

Abstraction hides **implementation details** and shows only **essential functionality**.

## Example

```java
abstract class Vehicle {

    abstract void start();
}

class Car extends Vehicle {

    void start() {
        System.out.println("Car starts with key");
    }
}
```

### Key Points

* Achieved using **abstract classes or interfaces**
* Reduces **complexity**
* Improves **security**

---

# 📊 OOP Concept Relationship

```
OOP
│
├── Encapsulation → Data Hiding
├── Inheritance → Code Reusability
├── Polymorphism → Multiple Behaviors
└── Abstraction → Hide Complexity
```

# 📘 Class in Java

## 🧾 Definition
A **class** is a template or blueprint used to create objects. It defines how an object will look and behave.

---

## 🏗️ Components of a Class

A class contains:

- **Variables (Properties / Data Members)**  
  → Used to store data  

- **Methods (Functions / Behaviors)**  
  → Used to perform actions  

---

## 🚗 Real-World Example: Car

Think of a **Car** as a class.

### 🧩 Properties (What it *has*)
- Color  
- Brand  
- Type  

> 👉 These are **variables** (data storage)

### ⚙️ Actions (What it *does*)
- `start()`  
- `brake()`  
- `accelerate()`  

> 👉 These are **methods** (perform actions)

---

## 🧠 Simple Understanding

| Concept    | Meaning               |
|------------|----------------------|
| Properties | What an object *has* |
| Actions    | What an object *does* |

---

## ⚡ Quick Summary

- A **class = blueprint**  
- **Properties = variables (has)**  
- **Actions = methods (does)**  

---

## 💻 Example Code

```java
class Car {
    String color;
    String brand;
    String type;

    void start() {
        System.out.println("Car is starting");
    }

    void brake() {
        System.out.println("Car is stopping");
    }

    void accelerate() {
        System.out.println("Car is accelerating");
    }
}
```
